       Qwen2.5-14B-Instruct  ({{ .Values.vllm.servedName }})
       Qwen2.5-VL-7B-Instruct ({{ .Values.vllmVl.servedName }})
     and pull the embedding model into Ollama:
       kubectl exec deploy/{{ .Release.Name }}-ollama -- ollama pull {{ .Values.ollama.embedModel }}
  2. Configure the Keycloak realm "{{ .Values.keycloak.realm }}": create the
     `{{ .Values.keycloak.realm }}` OIDC client, set the redirect URI to
       {{ .Values.global.appUrl }}/api/auth/oidc/callback
     map IdP groups -> roles (owner/admin/manager/writer/reviewer/viewer), and
     federate Azure AD / ADFS if required. Set OIDC_ISSUER (keycloak.issuerUrl)
     to a BROWSER-REACHABLE URL.
  3. The app runs `prisma migrate deploy` on start; the pgvector + pgcrypto
     extensions are created by the postgres init ConfigMap. Apply the HNSW
     index from prisma/sql/knowledge_chunks_index.sql once.
  4. Verify DENY-ALL egress and run the readiness checklist
     (docs/phase4-enterprise-self-hosted.md §8).
